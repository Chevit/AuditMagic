# CI Test Gate Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Run `pytest tests/ -v` on every push/PR and block release builds until tests pass.

**Architecture:** Two changes — a new `test.yml` workflow for CI (push/PR on all branches) and a `test` job prepended to `build.yml` with `needs: [test]` on all three build jobs. Both use `ubuntu-latest` + Python 3.14 + `requirements-dev.txt`.

**Tech Stack:** GitHub Actions, Python 3.14, pytest, ubuntu-latest runner

---

### Task 1: Create `.github/workflows/test.yml`

**Files:**
- Create: `.github/workflows/test.yml`

**Step 1: Create the file with the following exact content**

```yaml
name: Test

on:
  push:
  pull_request:

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.14'

      - name: Set up virtual environment
        run: python -m venv .venv

      - name: Install dependencies
        run: |
          .venv/bin/python -m pip install --upgrade pip
          .venv/bin/pip install -r requirements-dev.txt

      - name: Run tests
        env:
          QT_QPA_PLATFORM: offscreen
        run: .venv/bin/pytest tests/ -v
```

**Step 2: Verify YAML syntax is valid**

Run:
```bash
python -c "import yaml; yaml.safe_load(open('.github/workflows/test.yml')); print('OK')"
```
Expected: `OK`

**Step 3: Commit**

```bash
git add .github/workflows/test.yml
git commit -m "ci: add test workflow — run pytest on every push and PR"
```

### Task 2: Add `test` job to `build.yml` and gate all builds on it

**Files:**
- Modify: `.github/workflows/build.yml`

**Step 1: Add the `test` job**

In `.github/workflows/build.yml`, directly after the `jobs:` line (before `build-windows:`), insert the following job block:

```yaml
  test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.14'

      - name: Set up virtual environment
        run: python -m venv .venv

      - name: Install dependencies
        run: |
          .venv/bin/python -m pip install --upgrade pip
          .venv/bin/pip install -r requirements-dev.txt

      - name: Run tests
        env:
          QT_QPA_PLATFORM: offscreen
        run: .venv/bin/pytest tests/ -v

```

**Step 2: Add `needs: [test]` to `build-windows`**

Find the line:
```yaml
  build-windows:
    runs-on: self-hosted
```

Change to:
```yaml
  build-windows:
    needs: [test]
    runs-on: self-hosted
```

**Step 3: Add `needs: [test]` to `build-macos`**

Find the line:
```yaml

  build-macos:
    needs: [test]
    runs-on: macos-latest
```

**Step 4: Add `needs: [test]` to `build-linux`**

Find the line:
```yaml
  build-linux:
    runs-on: ubuntu-latest
```

```yaml

    needs: [test]
```

**Step 5: Verify YAML syntax is valid**

Run:

**Step 6: Verify the final structure of `build.yml`**

The `jobs:` section should now start with `test:`, and each of `build-windows`, `build-macos`, `build-linux` should have `needs: [test]` as the first key under their job name.

Run:

Expected: 3 lines, one for each build job.

**Step 7: Commit**

